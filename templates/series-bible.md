# Template · Series Bible · 剧集圣经（季度规划 + 连续性追踪）

> The series planning and continuity document, produced in phase 1 and updated every
> episode. Governs `modules/formats/series.md`.

## Part A · Season plan · 季度规划（once, iterable）

```
CONFIG 配置
  Total episodes N · per-ep runtime · per-ep structure · series type · season arc?
  · world complexity

SEASON PREMISE 季度前提 (one logline)

SEASON SYNOPSIS 季度梗概 (one page, 300–500 字, prose — see treatment.md)

PROTAGONIST SEASON ARC 主角季度弧光
  From A → to B · Lie:

BACKSTORY & WORLD 前史与世界观 (cross-section principle)
  Era / context:
  Protagonist backstory (esp. Ghost):
  Relationship history:
  World — three layers: surface rules / operating logic / underlying truth
  Cross-section choice: why Ep1 starts here

ARC-BUDGET 弧光预算 (draft)
  first 25% of eps: cumulative ≤15% · ≥1 false change + retreat (40–60%) ·
  65–85% acceleration zone

HIDDEN-LINE PLAN 暗线规划
  Line | starts (ep) | reveals (ep) | what it changes
```

## Part B · Episode outline map · 分集大纲（the critical step）

Per episode — pull the whole season through before writing any episode:

```
EP[X] 《title》
  Logline:
  Core dramatic action: goal → obstacle → result
  Tonal type: setup / light / tense / breath / pivot / pressure / collapse /
              dark-night / counter / climax
  Inner structure (timed): cold open (Xs) / setup (Xm) / development (Xm, ≥2 turns)
              / climax (Xm) / resolution + act-out (Xm) — each with plot+emotion tags
  Season connections: arc spent · hidden-line advanced · world layer released ·
              seeds planted (→ pay at EP_) · seeds paid · supporting dynamics
  Inter-ep: picks up from / hands to / pacing vs neighbors
```

## Part C · Continuity tracking · 连续性追踪（update every episode）

**C1 · Character state**
```
Character | state at episode end | key change | must carry to next ep
```
**C2 · Hidden-line progress**
```
Line | current progress | advanced this ep | next advance (ep)
```
**C3 · Plant / payoff register**
```
Seed | planted (ep) | scene | planned payoff (ep) | how | paid? ✅/❌
```
**C4 · Public-information list**
```
Info | public since (ep) | how | who knows | who doesn't
```

## Discipline
- Phase 1 finishes the map and self-checks it (`series.md` workflow) before any
  episode is written.
- Update C1–C4 immediately after each episode; read them before writing the next.
- On a continuity conflict, fix the new episode, not locked old ones.
- Checkpoint each episode (`memory-checkpoint.md`, series add-on).
