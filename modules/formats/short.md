# Format · Short Film · 叙事短片（5–10 分钟）

> A single core event told with density. Loads `00` + `01` + `02`.

## Traits
Runtime 5–10 min · 1–2 leads with full Want/Need/Arc · four-part structure · one
A→B arc · world legible in 30s · high information density — every scene advances
plot or reveals character.

## Core constraint
A short allows **one single core event** — no subplots, no diffusion of focus.
Every scene serves the same dramatic action. Default to realism unless the user
asks otherwise.

## Stage scaling
| Stage | Short |
|---|---|
| 1 Break-in | core dramatic action (Goal + Conflict) |
| 2 Synopsis | one paragraph (100–200 字), tone telegraphs the film |
| 3 Character | Want / Need / Arc + one contradiction |
| 4 Backstory | lean — diffuse into dialogue/visual clues; no massed flashback |
| 5 Structure | four-part + opening hook (first 15s) |
| 6 Breakdown | micro-actions + runtime + dual-track tag |
| 7 Writing | Fountain; visual + subtext |
| 7d Direction | the breakdown is where the film is made — spend effort here |
| 8 Doctor | full diagnosis + pacing |

## Four-part structure
```
ACT I  Setup       — opening hook (15s, an ordinary image with one detail "off")
                   — establish the normal — inciting incident (break the balance)
ACT II Development — obstacle 1 → obstacle 2 (must differ in kind) → midpoint
                     (false win/loss) → all-is-lost
ACT III Climax     — final confrontation → the key choice and the turn
ACT IV Resolution  — the new normal (arc complete) — show it, don't explain it
```

## Technique
- **Backstory delivery** — diffuse into dialogue and visual clues; flashback only
  if nothing else works.
- **Stretch & snap** — build expectation with repetition/slow pace/static frames,
  then break the balance with no warning in the shortest possible AV beat.
- **Ending** — show the post-arc new normal in one image; don't explain what the
  character learned.

## Self-checks
*Structure:* hook in first 15s? four parts clear? inciting incident and climax
clear? each part advances plot or reveals character? obstacle 2 differs from
obstacle 1? runtime budget reasonable (setup ~15%, dev ~40%, climax ~25%, res ~20%)?
*Scene:* any interior narration (fail)? told visually? dialogue too on-the-nose?
subtext and body language used? spoken, not written?

Trigger a memory checkpoint after the breakdown if the short exceeds 8 min or 8 scenes.
