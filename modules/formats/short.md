# Format · Short Film · 叙事短片（默认 5–15 分钟）

> A concentrated event or transformation told with density. Minimum format
> dependency: `00`; task routes add development/structure only when needed. A
> custom profile may scale runtime or structure.

## Traits

Usually 1–2 leads · one concentrated event/transformation · world becomes legible
quickly · high information density. Every scene changes situation, character, or
audience understanding.

Want/Need/Arc and a four-part structure are strong defaults for narrative shorts,
not hard gates for essay, mood, experimental, or character-free forms. QA marks
irrelevant dimensions `N/A` and evaluates the chosen form's actual progression.

## Core constraint

A short normally concentrates on **one core event**. Any subplot must directly
intensify or reinterpret it. Default to realism only when the Project Card does not
choose another mode.

## Stage scaling

| Stage | Short |
|---|---|
| 1 Core action | dramatic engine or explicit formal/emotional progression |
| 2 Synopsis | one paragraph (100–250 字), tone telegraphs the film |
| 3 Character | Want / Need / Arc + contradiction when character-driven |
| 4 Backstory/canon | lean; deliver only facts the screen needs |
| 5 Structure | four-part default + prompt opening question/image |
| 6 Scene map | Scene Cards + runtime + pacing + state deltas |
| 7 Screenplay | Fountain or an approved AV-script variant |
| 7d Direction | visual design is disproportionately important at this scale |
| 8 QA | format-aware hard gates + runtime/continuity closure |

## Four-part default

```text
ACT I  Setup       — designed opening; normal and pressure arrive together
ACT II Development — obstacles/reversals differ in mechanism or cost; midpoint
ACT III Climax     — decisive confrontation, image, or formal/emotional turn
ACT IV Resolution  — a new state shown rather than explained
```

Nontraditional shorts may replace the climax/arc with a legible audiovisual,
associative, or emotional progression. Record that choice in the Project Card so QA
does not apply an irrelevant conventional rubric.

## Techniques

- **Backstory delivery** — behavior, prop, space, relationship friction, or
  discovery; intentional flashback/VO only when it creates an audiovisual gain.
- **Stretch & snap** — build a readable pattern, then break it in the smallest
  decisive beat.
- **Ending** — land a new state or re-reading in an image/sound; do not explain the
  lesson.

## Self-check

- Does the opening create a concrete question or establish the form promptly?
- Does each unit turn plot, character, relationship, knowledge, or the governing
  audiovisual pattern?
- Do repeated obstacles differ in mechanism, cost, or meaning?
- Does runtime add up and leave the decisive turn enough room?
- Is every claimed inner shift playable/visible/audible in the chosen form?
- If dialogue/arc/climax is absent by design, does another progression carry the
  audience?

Create a Project State/checkpoint after the scene map for work over roughly eight
scenes or any multi-session short.
