# Format · Vertical Micro-Drama · 竖屏短剧

> An industrialized emotion product for vertical mobile platforms. Adapts a novel /
> outline / seed into shoot-ready episodes for live-action or 2D/3D/AI-generated
> pipelines. This is **one format among many** — the craft core (`00`) still
> applies; this module adds the genre's specialized engine.
> 短剧只是诸多格式之一；它在通用编剧内核之上叠加情绪工程引擎。

> **Output discipline note.** Micro-drama **batches** output (it does not gate every
> stage). Live-action delivery normally folds a lightweight per-clip shot note into
> the script instead of a separate shot list. For AI generation, that same note is
> the authoring view but must be normalized to the canonical shot-row fields before
> creating generation packets; the CSV may stay an intermediate. Speed is the
> product, but production has one unambiguous contract.

> **Platform note (v0.2).** Timing, word-count, monetization, and compliance numbers
> below are Filmwright house defaults/creative heuristics—not claims about a
> platform's current official rules. For a real commission, verify a dated platform
> pack from official sources using `modules/platforms/platform-pack-template.md`;
> that pack overrides these defaults.

## §1. Root cognition · 根认知
Working attention hypothesis: establish a question within ~3 seconds, create an
effective release/escalation within ~30 seconds, and change direction within roughly
one minute. Tune with actual project analytics; do not present these as universal
viewer law. Therefore: every page
has someone winning or losing (no "chatting"); every line is a knife, hook, or
bomb; every scene ends making the viewer want the next; pause at any second and
the viewer must *need* to know what happens next. **Adaptation ≠ porting** — keep
and intensify the source's payoffs/lines/set-pieces; compress everything that drags
into one action or one line.

## §2. Parameters · 参数锁定
platform + region + dated pack (if available) · aspect (9:16 default / 16:9) · production
form (live-action / 2D / 3D / AI-gen) · genre lane · target total runtime · total
episodes · clips/episode (6–8 std, 9–12 long) · per-episode runtime · output tier
(Lite / Standard / Studio). Episode estimate (three-track): effective conflict
units (1 full conflict ≈ 1 ep) + word calibration (3000–5000 字 ≈ 1 ep) + runtime ÷
per-ep. Lock, then execute without re-asking.

| Season length | Episodes | Growth turns | Rhythm |
|---|---|---|---|
| Short | ≤10 | ≥2 | three-act |
| Standard | 11–24 | ≥3 | four-act |
| Long | >24 | 4–5 | phased; reset the face-slap ladder each phase |

## §3. Adaptability gate · 可改编性闸门
Eight-dimension pre-judgment (1–5 each): topic pull · protagonist relatability ·
conflict density · visualization dividend · differentiation · compliance safety ·
production feasibility · long-line support. Grade A (adapt directly) / B (adapt
after shoring up weaknesses) / C (rescue-restructure first). **Auto-C veto:** no
clear protagonist goal · no stable main conflict · selling point is setting with
no event · only legible via heavy narration · high-risk content is the main draw.

## §4. Adaptation method — seven steps · 改编七步流
1. **Extract the non-negotiables** — set-pieces (by impact), quotable lines, payoff
   moments (tag the type), character anchors.
2. **Cut** — interior >3 lines → one action or one OS; setting exposition → an event;
   redundant minor roles → merge; dead routine → cut or a one-clip montage.
3. **Core driver line** — `protagonist + wants + blocked by + loses what on failure
   + deadline/pressure`. Can't write it → the spine is unclear, return to the gate.
4. **Re-order the timeline** — mark the strongest conflict as the EP01 open
   candidate; pull a fact forward if the source opens slow; fill any gap with an
   action/line/prop, not narration.
5. **Compliance scan** — tag R0–R3, give safe substitutes (§12).
6. **Spine & growth anchors** — arc start→end, growth-turn episode numbers, verify
   the causal chain.
7. **Quality pre-check** — topic pull, relatability, relational tension, antagonist
   functionality; propose fixes for any weak dimension.

Novel→script conversions: big interior → one micro-expression + one small action ·
"he felt angry" → ▲ his hand clenches under the table, knuckles white · monologue
motive → one OS (≤12 字) or a flash image · narration setting → a conflict event ·
"everyone shocked" → who stepped back, whose cup shattered, who knelt · transition
→ hard-cut to the next conflict.

## §5. Emotion engine · 情绪引擎
**Six-beat rhythm 钩-压-弹-炸-荡-断:** Hook (intrusion of crisis, first clip) →
Press (pressure/insult/threat, ≤30%, ≤2 consecutive clips) → Spring (protagonist's
first reaction) → Blast (the payoff — face-slap / crush / reversal, 30–40%) →
Ripple (aftershock + new threat, bury a fresh bomb) → Cut (break on an unfinished
beat). One full cycle per episode (two for a long episode). The Blast must match
the Press; the Cut must never resolve flat.

**Emotion meter 情绪值** (+5 ecstatic … 0 calm [banned in EP01] … −3 heavy press).
Anchors: first effective positive (≥+2) ≤ C02; main Blast (≥+4) ≤ C04 (≤C08 long);
peak ≥+4 each episode (low-ebb episode waived to ≥+2, ≤2 per season); episode end
+2 to +4 (special/suspense/betrayal episodes waived to +1/−1 *with* a strong hook
and a rebound by next episode's C04).

**Emotion-type coverage:** ≥2 types per episode (爽 / 期待 / 代入 / 心疼 / 宣泄 /
抚慰 / 共鸣); EP01 must build 代入感 (relatability); a 心疼/抚慰/共鸣 moment every
5 episodes. Build relatability with a **visualized hardship close-up** in the first
30s (show the poverty, don't say it).

## §6. Payoff arsenal · 爽点武器库
A 力量碾压 (A1 physical / A2 weak-beats-strong illusion / A3 escalating / A4
condescension / A5 turnabout) · B 身份反转 (B1 reveal / B2 object value / B3
relationship / B4 prophecy / B5 "you know who you're talking to") · C 打脸 (C1
ability / C2 evidence / C3 rule / C4 自食其果 / C5 domino) · D 情感爆发 (D1 "I don't
need you" / D2 someone stands up / D3 mass defection / D4 late apology) · E 智谋 (E1
turn the plan / E2 info-gap / E3 close the net / E4 verbal) · F 成长 (F1 breakthrough
/ F2 ideology / F3 choice / F4 inheritance).

**Arrangement:** EP01 main payoff includes A or B (suspense lane may use E2/C3/B3 +
a clear identity/rule gap; sweet-pet lane may use a "sweet-blast" ≥+4 + an A/B aux
≥+3). No identical subtype two episodes running. B-type and F-type each ≥1 per 5
episodes. D never carries an episode alone; E sparingly in the first 3; F main
payoff not before EP04. Cover ≥3 of the six classes per 3 episodes. No three ceiling
episodes in a row. Watch prop fatigue (DNA report / recording / black card / token).

## §7. Dialogue rules · 台词军规
House pacing defaults: regular line ≤12 字; golden line ≤14 字 (1–2/episode);
per-clip total 25–60 字 unless an actor read or dated platform pack changes it; ≤4
dialogue rounds; VO ≤3/season (EP01 ≤2, ≤15
字/line, only for time-jump or system prompt — never for backstory). **Golden first
line** instantly states who a character is. Sentence weapons: knife / hook / bomb
(calmest tone + most violent content) / reversal / definition / unfinished. Replace
profanity with **condescension** — "也配？" beats a slur, safer and harder. Silence
allowed after a crush (≤1/episode) with a signature small action.

## §8. Character · 角色（在 `00-core-craft.md` §2 之上）
**Crush signature 碾压签名** (contemptuous / silent / smiling / violent / elegant,
kept consistent) + **relatability anchor 代入感锚点** + **crack 反差裂缝** (one
vulnerability, revealed every 5 ep). Both signature *and* anchor built within the
first 2 clips of EP01. Voice fingerprint per role, consistent all season (mismatch
= broken character, fix it). Antagonist five things: wants what · why arrogant ·
key blind spot · favorite method · how the echo-slap pays it back. Antagonist
escalation: minion (1 ep) < mid-tier (≥2 ep) < big boss (≥5 ep, exits on a B-type).

## §9. Set-piece & beat · 场景与结构
**EP01 opens** (pick one): trial / violence / status-gap / flashback / wedding /
hospital-or-funeral / caught-by-coincidence. In 30s build crisis + contrast + enemy
+ rule + relatability; in 15s the viewer knows whose side to take. Scene rule: enter
with a task, leave with a bomb; EP01 ≤3 scenes (A-grade ≤2); environment amplifies
emotion. **Episode-end bombs T1–T15** (stronger enemy / identity about to surface /
the villain smiled / threat-no-action / new info / ally ambiguous / physical
suspense / pause-before-reveal / declaration / cost surfaces / pregnancy-bloodline /
betrayal caught / the unlikeliest yields / call-or-message / growth cost) — no same
number two episodes running; last line is an image or an unfinished sentence.

**Coincidence engine:** plant the nail 1–2 clips early → trigger at the most
dramatic moment → chain to the whole room. **Face-slap ritual** (public): audience
(incl. a villain ally) + echo (slap how they bragged) + ≥2 lines of villain
collapse (private: echo + collapse only). Face-slap ladder: physical → evidence/
identity → social → ultimate, scaled by season length, reset per phase in long
projects.

## §10. Main-line architecture · 主线架构
**Three locks 三锁:** suspense lock (core question, EP01, a piece every 3–5 ep) ·
relationship lock (EP01–02, an event every 2–3 ep) · destiny lock (EP01 hint, via
growth turns). **Causal chain:** each Blast leaves ≥1 irreversible consequence;
EP01–03 events echo at EP08–10 (short season: EP01–02 echoes 2–3 ep before finale);
the big boss's scheme runs in the undertow from EP01. Reversal density: ≥1 per 3
episodes, types rotating (info / relationship / result). Subplots ≤2 (short/standard)
or ≤3 (long), each causally tied to the main line and merging at a node. Paywall
node usually EP03–05: identity about to surface but not / biggest slap imminent but
not / relationship about to flip but not.

## §11. Four ledger views · 四账本视图（信息管理）
Who knows what, when it detonates. Store canonical facts in Project State; refresh
these task-focused views on every revision:
- **Character** — name｜function｜surface id｜true id｜goal｜hidden motive｜tension｜
  voice｜crack｜growth nodes｜exit.
- **Information** — name｜known｜unknown｜hidden from whom｜audience-first?｜reveal ep｜
  reveal method.
- **Causality** — seed ep｜seed event｜ferment｜result ep｜result event｜affects whom.
- **Prop** — name｜first seen｜state｜holder｜function｜next trigger ep｜final state.

## §12. Safety, rating & platform rail · 安全、分级与平台护栏

Use host safety, applicable law, the Project Card's audience/rating, and the current
dated platform pack. Do not turn culturally vague or obsolete platform language
into a universal creative rule.

Risk-map each project for: minors; sexual/intimate content; violence/gore;
self-harm; hate/harassment; dangerous/criminal instruction; weapons/stunts/vehicles/
animals; coercive relationships; living-person likeness/claims; copyright/brand;
and region/platform-specific restrictions. Escalate uncertainty instead of
pretending clearance.

Safer depiction can be more precise: a public strike → raised hand, reaction, then
aftermath; graphic injury → balance loss, breath, prop, and consequence; coercive
control framed as romance → make harm legible and preserve agency. Ask four value
questions: does the win rely only on money/power? is the antagonist bad only by
label? is coercion romanticized? does the story preserve a positive value anchor?

## §13. Production adaptation · 成片形态适配
| Form | Image | Dialogue | Action/space |
|---|---|---|---|
| **Live-action** | actable, blockable, continuous; write the *result* of spectacle, don't lean on FX | spoken, natural, biggest subtext space | strict physical logic; violence/intimacy via implication |
| **2D** | exaggerated expression, symbolic action; solid/gradient backgrounds for emotion | sharpest, most slogan-like; high golden-line density | payoffs via pose + expression + prop reveal; text FX as amplifier |
| **3D** | spatial depth, light layers, position = power; step-in/pause as pressure | slightly slower, more silence allowed | room to move; control scene complexity; particle/shatter as amplifier |
| **AI-gen** | embedded clip note → canonical shot row → `production/ai-video.md` packet | normalized 13-field row is required for production, even when CSV is not delivered | asset IDs, endpoint continuity, concrete actions, verified adapter |

Production economy: ≤5 recurring locations/season; reuse an EP01 location within 3
episodes; one-time full appearance description per character + a visual tag-object.

## §14. Output · 输出（Studio, 3 batches）
1. params + adaptability + adaptation-decision summary → **wait for confirm**.
2. project anchor + four ledgers + season episode plan + EP01 emotion design + EP01
   full script (Fountain or the platform's standard format) → continue.
3. self-check table + delivery attachments (character/scene/prop sketches, inter-episode
   handoff) + version log → complete.
Lite = anchor + episode skeleton + EP01 emotion design. Standard = anchor + light
ledgers + episode plan + EP01 full script. **CONTINUE mode:** start from the emotion
design, pick up the previous episode's hook in 3 seconds, no recap; keep info/growth/
locks consistent with the handoff.

## §15. Self-check (hard items) · 集级自检
3s hook · 15s side-taking · first positive ≤C02 (≥+2) · main Blast ≤C04 · main payoff
has A/B · ≥1 aux release · press ≤2 consecutive clips · press ≤30% · peak ≥+4 ·
end +2..+4 · no tepid/explaining lines · no accidental interior narration (only
intentional VO within the declared limit) · ≥1 irreversible
main-line event · concrete imagery · dual-track tag per scene. Report only
`[⚠ failed]` and `[waiver]` items; if all pass, write "self-check passed."

## Rewrite triggers · 质量红线
tool-puppet dialogue · teleporting romance · mechanical face-slaps · cost-free power-up
· plastic empathy (telling, not showing the pain) · interior-narration leak · a
character explaining setting. Any one → rewrite that segment.
