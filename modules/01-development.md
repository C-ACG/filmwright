# Module 01 · Development · 开发（选题 · 立意 · 人物 · 世界）

> The front of the pipeline (stages 1–4). How to go from nothing — or from a raw
> seed — to a load-bearing premise, a character with internal tension, and a world
> ready to be cut into.
> 流水线前段：从零或从粗糙种子，得到能承重的设定、有内在张力的人物、可被切入的世界。

---

## §1. Ideation toolkit · 选题工具库（内部武器，不展示给用户）

When the user has no concept yet, **do not show them this menu.** Listen, infer
which path their vagueness belongs to, then sharpen the concept through natural,
specific, generative questions. The user should experience "you helped me find a
great idea," not "here is a methodology, pick one."

**Five entry paths · 五条选题路径** — read the signal, pick the tool:

| Path | Signal | Method |
|---|---|---|
| **From theme** | an abstract proposition, social issue, "I want a story about X" | turn the abstract into a concrete dramatic action — "about memory" is not a story; "a man losing all memory in 24h must choose which one to keep" is. Ask: who, in what corner, gets forced to choose? |
| **From character** | a contradictory person ("a swim coach terrified of water") | mine the contradiction — it *is* the seed of conflict. Ask: what does this person most fear facing? force them to face it. |
| **From space** | a place, a closed environment, a special world | space = rules. Ask: who's on top, who's below? what's scarce? what's forbidden here? (See *Grand Budapest*, *The Platform*, *Snowpiercer*.) |
| **From relationship** | tension between two people | conflicting Wants. Ask: each one's Want? why can't both be satisfied? what situation pushes it to the extreme? |
| **From collision** | a few keywords, a vibe, "combine X and Y" | use the five combination tools below. |

**Five combination tools · 概念组合**（also usable to re-sharpen any tepid premise):
- **Graft** — A's mechanism + B's domain. (Game respawn + layoffs → "the laid-off can re-spawn, but each respawn halves the salary.")
- **Anachronism** — a person/tech/system in a time/place it doesn't belong. (ChatGPT + imperial exam.)
- **Metaphor made physical** — an abstract anxiety as a mandatory physical rule. ("Your lamp must be brighter than your neighbor's, or the power cuts out.")
- **Ironic juxtaposition** — two superficially alike, fundamentally opposed things in one frame. (Pet care + slaughterhouse next door.)
- **Rule inversion** — flip a taken-for-granted rule and follow the consequence. ("The harder you work, the less you own.")

**Operating flow:** listen → infer path (user need not know paths exist) → deepen
with the matching tool → produce 3–5 concrete options → never expose the
methodology. If the user is fully blank, prompt with: a recent absurd
news/image? a real rule you'd most love to invert? a place that already feels like
a story? two unrelated things that'd be fun together?

## §2. Premise & logline · 设定与一句话故事

Two tests every premise must pass (borrowed from the concept-film discipline):
1. **One sentence.** Can the core be said in a single clear sentence?
2. **Five seconds.** Can it be shown — not explained — in five seconds of footage?

The logline shape: `who + wants + opposed by + at the cost of + under what pressure`.
If you cannot write this line, the spine is unclear — go back to stage 1.

> 真千金被误认为坏人，必须在继承权生效前拿回母亲的股份，否则同时失去身份、婚姻与母亲死亡的真相。
> (who · want · pressure/deadline · what's at stake)

## §3. Character build · 人物搭建

Follow `00-core-craft.md` §2 for the model. This stage produces the deliverable:

**Short form output:**
```
Name 角色：
Want 外在需求：
Need 内在欲望：
Arc 弧光：A → B
Bio key points 小传关键点：
Contradiction 矛盾特征：outer vs inner
```

**Feature / series output:** add `Ghost / Lie / Flaw`, the B-story character (who
embodies the opposite side of the thematic argument), and supporting-cast function
tags. Each supporting character carries **at least one** of:

| Function 功能 | Role |
|---|---|
| **Mirror** 镜像 | walks the road the protagonist didn't — "who they'd become with a different choice" |
| **Catalyst** 催化 | the external force that drives change — mentor, tempter, challenger |
| **Theme** 主题 | embodies a stance in the thematic argument (usually the B-story role) |
| **Pace** 节奏 | comedic relief / emotional ballast — a place for the audience to breathe |

A character with no function should be merged or cut. Two characters with the same
function merge into one. Supporting voices must read distinct enough that, with
names hidden, the audience knows who is speaking.

**Ensemble add-on:** for 3+ co-leads, build a **relationship matrix** (state
between each pair), a POV-switching rhythm, and a shared theme all Wants point at.

## §4. World as cross-section · 前史与世界观

Build the world *before* outlining (stage 4 < stage 5). Output:

```
Era / context 时代背景：what age, what social mood; which feature shapes the character's plight
Protagonist backstory 主角前史：from birth to story-start, key events — especially the Ghost
Relationship history 关系既往：where each key relationship stood before frame one
World rules 世界观规则（if applicable）：the three-layer model below
Cross-section choice 横截面选择：why start *here*? what makes "now" the only place to begin?
```

**Three-layer world model (where complexity warrants):**
```
Surface rules 表层规则   — how the world plainly works           (revealed early)
Operating logic 运行逻辑 — why the world is this way             (revealed mid)
Underlying truth 底层真相 — the world's ultimate face            (revealed at climax)
```
Each layer is revealed through character action and discovery — never through
narration, titles, or a character sitting down to "explain." Each reveal must
change how the audience reads everything before it.

**Backstory delivery rule:** diffuse backstory into dialogue and visual clues.
Avoid massed flashback or VO exposition; use flashback only when nothing else works.
